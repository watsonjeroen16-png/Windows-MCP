# Getting Every Credential Kaizi Needs (Explain-It-Like-I'm-5 Edition)

This is the beginner-friendly, click-by-click version. If you've never signed
up for a developer account or an API key before, start here. Every account
below has to be created by **you personally** — a real name, a real email, and
usually a real credit card — because these companies are handing out access
to send text messages, call an AI, host a server, or publish an app under
your name. Nobody else can do this step for you.

For the condensed checklist (what's done, what's blocked, cost at a glance),
see `kaizi/docs/DEPLOYMENT-READINESS.md`. For the full deployment mechanics
once you have these credentials, see `kaizi/server/DEPLOYMENT.md` (backend)
and `kaizi/app/DEPLOYMENT.md` (mobile app) — this doc won't repeat those
steps, it only tells you how to get the keys those guides ask for.

## How this is ordered

**Part 1** is everything you need to get Kaizi actually running — first on
your own computer/phone, then live on the internet for real users. Do these
first.

**Part 2** is everything you only need when you're ready to put Kaizi in the
Apple App Store or Google Play Store. You can skip Part 2 entirely until
launch day — the app runs fine without it.

# Part 1 — Get a working app running

## 1. Anthropic API key (the AI brain behind your companion)

**What it is:** A password-like string that lets Kaizi's server talk to
Claude, the AI model that writes your companion's replies in the chat
screen.

**Why Kaizi needs it:** Without it, the companion doesn't actually think —
it just recites one of a handful of pre-written canned sentences per
personality (see `kaizi/server/src/services/claude-chat.ts`). With the key,
every reply is freshly generated by Claude, in your companion's voice. This
is the difference between a real companion and a scripted one.

**Sign up here:** https://console.anthropic.com

**Click-by-click:**
1. Go to https://console.anthropic.com and create an account (this is a
   separate account system from a personal claude.ai subscription — even if
   you already pay for Claude, you still need to sign up here).
2. Once logged in, go to **Settings → Billing** and add a payment method.
   This is "pay as you go" — you're billed for what the app actually uses,
   there's no flat subscription fee.
3. Go to **Settings → API Keys → Create Key**. Give it a name like
   `kaizi-production` so you can recognize it later.
4. Copy the key immediately — it's shown to you exactly once and looks like
   `sk-ant-...`. If you lose it, you'll have to make a new one.

**Where it goes:**
- Local development: `kaizi/server/.env`, as `ANTHROPIC_API_KEY=sk-ant-...`
  (copy `kaizi/server/.env.example` to `.env` first if you haven't).
- Production (Railway): the `ANTHROPIC_API_KEY` variable on the
  `kaizi-server` service, per `kaizi/server/DEPLOYMENT.md` §3.

## 2. Twilio (phone number verification + text messages)

**What it is:** Twilio is the company that actually sends text messages on
Kaizi's behalf — both the "here's your 6-digit code" verification text when
someone signs up, and the daily check-in/encouragement texts from the
companion.

**Why Kaizi needs it:** Kaizi verifies every new user by texting them a
code (so people can't sign up with fake or stolen phone numbers), and the
companion relationship partly lives in SMS (morning/evening messages). Both
of those require an account that can send real text messages. Without it,
Kaizi runs in a safe "pretend" mode where the code is always `000000` and
texts are only logged, never actually sent.

**Sign up here:** https://www.twilio.com/try-twilio

**Click-by-click:**
1. Go to https://www.twilio.com/try-twilio and sign up with your email. You
   will need to verify your own phone number as part of signup.
2. You land on the **Console Dashboard**. Right there you'll see **Account
   SID** (starts with `AC...`) and **Auth Token** (click the eye icon to
   reveal it). Copy both somewhere safe — the Auth Token is a secret,
   treat it like a password.
3. Buy a phone number that can send SMS: left sidebar → **Phone Numbers →
   Manage → Buy a number**. Pick one in your country with the "SMS"
   capability checkbox available. Trial accounts get a small amount of free
   credit to do this.
4. Set up phone verification: left sidebar (or **Explore Products**) →
   **Verify → Services → Create new Service**. Name it something like
   "Kaizi". After creating it, open the service and copy its **Service
   SID** — starts with `VA...`.

**Where it goes** (all four are required together — Kaizi falls back to
mock/pretend mode if even one is missing):
- `TWILIO_ACCOUNT_SID` — the `AC...` value from step 2.
- `TWILIO_AUTH_TOKEN` — the Auth Token from step 2 (keep secret).
- `TWILIO_VERIFY_SERVICE_SID` — the `VA...` value from step 4.
- `TWILIO_MESSAGING_FROM` — the phone number you bought in step 3, written
  like `+15551234567` (this is called "E.164 format" — just the `+`, the
  country code, and the digits, no spaces or dashes).

Locally these go in `kaizi/server/.env`. In production they go in Railway's
**Variables** tab on the `kaizi-server` service — see
`kaizi/server/DEPLOYMENT.md` §3 for the exact table.

*Note: while your Twilio account is still on a "trial", Twilio may only let
you text phone numbers you've manually verified as your own. That's normal —
it goes away once you add a payment method and Twilio upgrades you off the
trial.*

## 3. Railway (where the server actually lives)

**What it is:** Railway is a hosting company — it's the computer in the
cloud that runs Kaizi's backend server 24/7 so the app works for users
anywhere, not just on your laptop.

**Why Kaizi needs it:** Your phone can't talk to a server running on your
personal computer once you close the laptop lid. Railway keeps the server
always on, at a real web address (like
`kaizi-server-production.up.railway.app`), which is what the app's
`EXPO_PUBLIC_API_URL` setting points to.

**Sign up here:** https://railway.app

**Click-by-click:**
1. Go to https://railway.app and click **Login**, then choose **Login with
   GitHub** (easiest, since the code already lives on GitHub).
2. That's it for account creation — Railway gives you a free trial amount
   of usage, then a small monthly cost once you outgrow it (a single small
   server like Kaizi's is inexpensive).
3. The actual project setup (creating the service, connecting the database,
   entering environment variables, deploying) is a longer process — follow
   `kaizi/server/DEPLOYMENT.md` start to finish once you're logged in. That
   guide is already written and tested; this section is only about creating
   the account itself.

**Where it goes:** There's no single "key" to paste anywhere for Railway
itself — you log into their dashboard directly with your GitHub account and
manage everything there, as described in `kaizi/server/DEPLOYMENT.md`.

*While you're in Railway setting things up, that guide will also ask you to
generate a `SESSION_SECRET` value yourself (one command on your own
computer, no signup needed) — that's covered fully in
`kaizi/server/DEPLOYMENT.md` §3, not repeated here since it isn't a
third-party credential.*

## 4. EAS / Expo account (builds the app into something installable)

**What it is:** Expo is the toolkit Kaizi's app is built with. EAS
("Expo Application Services") is Expo's cloud build service — it takes the
app's code and turns it into an actual installable app file (like an
`.apk` for Android or a build for iOS), because that process needs a Mac
and specialized tools most people don't have lying around.

**Why Kaizi needs it:** Without an EAS account, there's no way to produce
an installable build of the app at all — not for testing on your own phone,
and not for the app stores later.

**Sign up here:** https://expo.dev/signup

**Click-by-click:**
1. Go to https://expo.dev/signup and create a free account.
2. On your own computer, install the command-line tool and log in:
   ```
   npm install -g eas-cli
   eas login
   ```
3. From inside the project, link it to your new account:
   ```
   cd kaizi/app
   eas init
   ```
   This will ask which Expo account/organization owns the project (pick the
   one you just created) and confirm the project's slug, which is already
   set to `kaizi` in `kaizi/app/app.json`. Running this command
   automatically writes a new `extra.eas.projectId` field into `app.json` —
   you don't need to type anything in yourself, just answer the prompts.

**Where it goes:** Nothing to copy-paste — `eas login` stores your session
on your computer, and `eas init` writes the project ID directly into
`kaizi/app/app.json` for you. From then on, the build commands in
`kaizi/app/DEPLOYMENT.md` §2 (`eas build --profile development`, etc.) just
work.

**Cost:** Free for a normal amount of building. Expo sells paid tiers only
if you need many builds running at the same time.

# Part 2 — Only needed when you're ready to submit to the App Store / Play Store

Do not start these until the app itself is finished and approved for
launch — see `kaizi/docs/DEPLOYMENT-READINESS.md` for why a store
submission is currently on hold. These two accounts cost real money and
involve identity verification, so there's no benefit to getting them early.

## 5. Apple Developer Program (needed to put Kaizi on iPhones)

**What it is:** A paid membership from Apple that lets you distribute an
app through TestFlight (for testers) and the App Store (for everyone).
Apple requires this before any iOS app — even one shared privately — can
run on a phone that isn't physically plugged into a developer's Mac.

**Why Kaizi needs it:** There is no way around this one if you want an
iPhone version of Kaizi. It's Apple's rule, not a technical limitation.

**Sign up here:** https://developer.apple.com/programs/enroll/

**Click-by-click:**
1. Go to https://developer.apple.com/programs/enroll/. You'll need an
   Apple ID (the same kind of account you use for iCloud/the App Store — if
   you don't have one, it'll prompt you to create one first).
2. Choose to enroll as an **Individual** or an **Organization**. Individual
   is faster (usually approved within a day). Organization shows your
   company's name as the app publisher but requires Apple to verify your
   business (a D-U-N-S number lookup, sometimes a phone call) — this can
   take anywhere from a day to a couple of weeks, so plan ahead if you want
   the company name to appear.
3. Pay the **$99/year** fee with a credit card as part of enrollment.
4. Once approved, note two things you'll need later:
   - **Apple Team ID** — found at developer.apple.com → **Account** →
     **Membership Details**.
   - Log into **App Store Connect** (https://appstoreconnect.apple.com) →
     **My Apps** → **+** → **New App**, and create Kaizi's app listing
     using the same bundle identifier already set in `kaizi/app/app.json`
     (`ios.bundleIdentifier`, currently `com.kaizi.app`). Once created,
     go to that app's **App Information** page and note the **Apple ID**
     field — a numeric value, this is the "App Store Connect App ID".

**Where it goes:** All three values below go into
`kaizi/app/eas.json`, in the `submit.production.ios` block, replacing the
existing `REPLACE_WITH_*` placeholders:
- `appleId` → the email address of the Apple ID you enrolled with.
- `ascAppId` → the numeric App Store Connect App ID from step 4.
- `appleTeamId` → the Apple Team ID from step 4.

Full submission steps (running `eas submit --platform ios`) are already
written up in `kaizi/app/DEPLOYMENT.md` §3.

## 6. Google Play Console (needed to put Kaizi on Android phones)

**What it is:** Google's equivalent of the Apple Developer Program — a
developer account that lets you publish apps to the Google Play Store.

**Why Kaizi needs it:** Same story as Apple: there's no way to distribute
an Android app through Google Play without this account.

**Sign up here:** https://play.google.com/console/signup

**Click-by-click:**
1. Go to https://play.google.com/console/signup and sign in with a Google
   account.
2. Pay the **$25 one-time fee** (not yearly — you only pay this once,
   ever). Google may also ask you to complete identity verification as
   part of newer account requirements — follow their prompts.
3. In the Play Console, click **Create app**, name it "Kaizi", and use the
   same package name already set in `kaizi/app/app.json`
   (`android.package`, currently `com.kaizi.app`).
4. Create a service account so builds can be submitted automatically
   without you manually clicking "upload" every time:
   - Play Console → **Setup → API access** → link (or create) a Google
     Cloud project.
   - Still in that screen, click through to create a **service account**
     (this opens Google Cloud Console → **IAM & Admin → Service
     Accounts → Create Service Account**). Give it any name, e.g.
     `kaizi-eas-submit`. No special Cloud roles are needed there.
   - Back in Play Console's **API access** page, find your new service
     account and grant it the **Release manager** permission for the
     Kaizi app.
   - Back in Google Cloud Console, open the service account → **Keys** →
     **Add Key → Create new key → JSON**. This downloads a `.json` file —
     this file *is* the credential, treat it like a password.

**Where it goes:**
- Save the downloaded JSON file as
  `kaizi/app/google-play-service-account.json` — this exact filename is
  already referenced in `kaizi/app/eas.json`'s
  `submit.production.android.serviceAccountKeyPath`, so no config changes
  needed once the file is in place.
- **Never commit this file to git.** It's already listed in
  `kaizi/app/.gitignore`, so a normal `git add`/`git commit` won't pick it
  up — but double-check before pushing if you ever `git add -A`.

Full submission steps (`eas submit --platform android`) are already written
up in `kaizi/app/DEPLOYMENT.md` §4. Note: brand-new Play developer accounts
must go through a closed testing track with a minimum number of testers
before Google allows a full public release — budget a few days to a couple
of weeks for this the first time.

## Quick reference: every credential and where it lives

| # | Credential | Get it at | Goes into |
|---|---|---|---|
| 1 | `ANTHROPIC_API_KEY` | console.anthropic.com → Settings → API Keys | `kaizi/server/.env` (local) or Railway Variables (prod) |
| 2 | `TWILIO_ACCOUNT_SID`, `TWILIO_AUTH_TOKEN`, `TWILIO_VERIFY_SERVICE_SID`, `TWILIO_MESSAGING_FROM` | twilio.com/try-twilio → Console + Verify | `kaizi/server/.env` (local) or Railway Variables (prod) |
| 3 | Railway account | railway.app → Login with GitHub | N/A (dashboard login, not a pasted key) |
| 4 | EAS/Expo account + `extra.eas.projectId` | expo.dev/signup, then `eas init` | `kaizi/app/app.json` (auto-written by `eas init`) |
| 5 | Apple `appleId` / `ascAppId` / `appleTeamId` | developer.apple.com/programs/enroll, appstoreconnect.apple.com | `kaizi/app/eas.json` → `submit.production.ios` |
| 6 | Google Play `serviceAccountKeyPath` (JSON file) | play.google.com/console/signup → API access | `kaizi/app/google-play-service-account.json` (referenced by `kaizi/app/eas.json` → `submit.production.android`) |

If anything in this doc ever disagrees with `kaizi/server/DEPLOYMENT.md` or
`kaizi/app/DEPLOYMENT.md`, those two files are the source of truth for exact
deployment mechanics — this doc exists purely to make the *signup* steps
approachable for a first-timer.
